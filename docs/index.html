How This Site Was Built
This isn't a mockup. The Figma file below was designed entirely by running figma-kit commands from the terminal — then the same design became this marketing page.
1

Created a Figma file via MCP

Used create_new_file from the Figma MCP server to create "figma-kit demo" in our team workspace.

2

Set up 3 pages with figma-kit

Ran figma-kit page create and page rename to create Landing Page, Components, and Design System pages.

3

Built the hero section

Used node create frame, fx mesh, fx accent-bar, text create, ui button, and ui badge to compose the hero.

4

Added 3 glass feature cards

Ran card glass -t noir --title "..." --desc "..." three times with different content. Each card got fx glow underneath.

5

Created 19 UI components

Page 2 showcases every UI primitive: buttons (5 variants), badges (4 colors), toggles, checkboxes, radios, tabs, dropdowns, table, and more.

6

Generated the design system

One command: figma-kit ds create -t noir. Creates color swatches, type scale specimens, and spacing bars — all auto-layout.

7

Created Figma variables

ds variables-create -t noir --name "Noir Tokens" pushed 11 color tokens into Figma's native variable system.

8

Exported CSS tokens

export tokens --format css -t noir prints clean :root CSS custom properties. No manual extraction needed.

9

Ran QA on all 3 pages

qa checklist -t noir walks every node checking for empty names, tiny targets, and tight spacing. 258 nodes, 0 issues.

10

Turned the Figma design into this site

The colors, fonts, layout, and copy on this page are a 1:1 match of the Figma file. Scroll down to explore the live embed.
